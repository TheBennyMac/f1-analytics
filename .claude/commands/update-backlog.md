# /update-backlog

Review the current state of the project and update BACKLOG.md to reflect:

1. **Mark completed items** — read the recent git log (`git log --oneline -20`) and tick off any backlog items that have been delivered.
2. **Add new items** — based on the current conversation, any new features, bugs, or ideas discussed that aren't yet in the backlog should be added under the correct MoSCoW priority and version milestone.
3. **Promote items** — if any "Could Have" items have become more urgent based on what we've learned, move them up.
4. **Update the current version** — if we've started a new release branch, update the "Current Version" header.

When done, summarise the changes made in 2–3 bullet points. Do not rewrite the whole file unless explicitly asked.
